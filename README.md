# MobileNet4DFR: Enhanced Real-Time Deepfake Detection

This repository contains the enhanced implementation of the deepfake detection architecture originally inspired by the CVPRW 2024 paper *"Faster Than Lies: Real-time Deepfake Detection using Binary Neural Networks"*. 

In this upgraded version, we address the practical inference bottlenecks of Binary Neural Networks (BNNs) on standard hardware by introducing **MobileNetV2** as the core backbone. Furthermore, we augment the feature extraction pipeline with **Gabor Filters** and a **Squeeze-and-Excitation (SE) Attention Block** to dynamically weight multi-modal input channels.

## 🚀 Key Improvements & Novelties

1.  **Lightweight Backbone (MobileNetV2):** Replaced the BNext architecture with MobileNetV2. While BNext offers theoretical FLOP reductions, MobileNetV2 provides **true real-time inference** on standard GPUs and edge devices without requiring specialized XNOR hardware accelerators.
2.  **Multi-Modal Inputs (RGB + FFT + LBP + Gabor):** * **FFT (Fast Fourier Transform):** Captures high-frequency artifacts and anomalous noise patterns generated by diffusion models.
    * **LBP (Local Binary Patterns):** Extracts micro-texture anomalies.
    * **Gabor Filters:** Introduced to capture multi-oriented edge and texture responses, highly sensitive to subtle facial interpolations (e.g., around eyes and wrinkles).
3.  **Squeeze-and-Excitation (SE) Adapter:** Before feeding the concatenated 6-channel input (3 RGB + 1 FFT + 1 LBP + 1 Gabor) into the backbone, an SE Block dynamically recalibrates channel-wise feature responses. The model *learns* which modality (color, frequency, or texture) is most critical for each specific image.

## 🧠 Architecture Overview

```text
Input Image (RGB) 
 ├──> FFT Extraction ───┐
 ├──> LBP Extraction ───┼──> Concat (6 Channels) ──> [ SE-Block ] ──> Conv2D (3 Channels) 
 └──> Gabor Filters  ───┘       (Attention)        (Compress)

                                                      │
                                                      ▼
                                              [ MobileNetV2 ]
                                                      │
                                                      ▼
                                           [ Linear Classifier ] ──> Real / Fake
```

## 🛠️ Setup & Installation

**1. Clone the repository and navigate to the directory:**
```bash
git clone [https://github.com/your-username/MobileNet4DFR.git](https://github.com/your-username/MobileNet4DFR.git)
cd MobileNet4DFR
```

**2. Set up the Conda environment:**
To ensure reproducibility and avoid dependency conflicts, create an isolated environment:
```bash
conda create --name dfad python=3.10
conda activate dfad
```

**3. Install dependencies:**
Install PyTorch (ensure your CUDA version matches, e.g., CUDA 12.x for modern RTX GPUs) and other required libraries:
```bash
pip install torch torchvision torchaudio --index-url [https://download.pytorch.org/whl/cu121](https://download.pytorch.org/whl/cu121)
pip install -r requirements.txt
pip install albumentations  # For advanced data augmentation
```

## 📂 Dataset Preparation

This project supports evaluations on **CIFAKE**, **COCOFake**, and **DFFD**. 

1. Create a `datasets` folder in the root directory.
2. Download and extract your datasets.
3. Your directory structure should look like this:
```text
MobileNet4DFR/
├── datasets/
│   ├── cifake/
│   │   ├── train/
│   │   └── test/
│   ├── coco2014/
│   └── fake_coco/
├── configs/
├── model.py
├── train.py
└── test.py
```

## ⚙️ Configuration & Training

**1. Generate Configuration Files:**
Run the script to generate `.cfg` files for different datasets and ablation studies.
```bash
python generate_configs.py
```
*(Make sure to update the dataset paths in your generated `.cfg` files if they differ from the default `../../datasets/...`)*

**2. Train the Model:**
To train the enhanced model (ensure your config file has `add_gabor_channel: True` and uses the MobileNet backbone), run:
```bash
python train.py --cfg configs/results_cifake_mobilenet_unfrozen.cfg
```
* The training process uses **Weights & Biases (Wandb)** for logging. You can log in with `wandb login` to track your Accuracy, Loss, and AUC in real-time, or choose option `3` to run offline.
* The best model checkpoint (`.ckpt`) will be automatically saved in the `DFAD_CVPRW24/` run folder based on Validation Accuracy.

## 🧪 Testing & Evaluation

To evaluate your trained model on the test set, create a `weights/` folder, place your best `.ckpt` file inside, and run:

```bash
python test.py --cfg configs/results_cifake_mobilenet_unfrozen.cfg
```
The script will output the **Accuracy**, **AUC**, and computational metrics (FLOPs/Parameters).

## 📊 Cross-Dataset Evaluation (Robustness Test)
To prove the model's generalization capabilities, you can evaluate the CIFAKE-trained checkpoint directly on the COCOFake validation sets by pointing the `--cfg` to a COCOFake config while keeping the `weights_path` pointed to your CIFAKE weights.
